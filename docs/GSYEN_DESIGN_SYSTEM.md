# GSYEN 疆域设计系统

本文档记录 GSYEN 当前的完整设计语言：世界观、两块设计、视觉识别、板块 Logo、信息流系统、组件规则和演化路线。

它不是一次性说明书，而是后续所有 UI 修改的判断基准。任何页面进入“疆域设计”，都必须先落到这里的语义和色度规则里，再去调整具体像素。

## 0. 一句话

**疆域不是聊天软件，也不是门户网站。疆域是一个个人信息疆域的工作坊系统。**

它的核心感觉应该是：

- 干净，但不空。
- 克制，但有力量。
- 信息密集，但不嘈杂。
- 工业化，但保留手工坊的温度。
- 黑色用于定位，金色用于信号，纸白用于承载，灰阶用于呼吸。

## 1. 两块设计

疆域现在有两块设计，不应该混成一种皮肤。

### 1.1 外部疆域：Frontier Entrance

代表页面：`LandingHero`

这是用户第一次看见疆域的地方。它不是工作台，而是“进入疆域”的仪式。

| 项 | 设计语义 |
|---|---|
| 背景 | 深黑网格，像尚未被开垦的信息空间 |
| Logo | 老爷车，代表人类第一次把生活半径推远 |
| 品牌字 | 疆域 + GSYEN，像碑铭，不像 SaaS 标题 |
| 动效 | 打字机式标语、轻微出现，不做炫技 |
| 按钮 | 进入工作坊、下载平台、GitHub，低亮度，不抢品牌中心 |
| 色彩 | 黑、白、极弱蓝、极少金 |

这一块的目标是“世界观成立”。

它可以更戏剧化、更暗、更像一扇门。它不需要展示复杂功能，也不需要看起来像后台系统。

### 1.2 内部疆域：Atelier Workbench

代表页面：灵阁、邮件、看板、日历、账簿、密钥、PRISM。

这是用户真正工作的地方。它不是入口海报，而是一个可长期停留、反复操作的信息工作台。

| 项 | 设计语义 |
|---|---|
| 背景 | 纸白、羊皮纸、低饱和暖灰 |
| 顶部 | 品牌铭牌 + 模块导航 |
| 中段 | 命令轨道、Pulse 信号带、模型状态 |
| 主体 | 当前模块工作区 |
| 底部 | 安全/本地/同步状态 |
| 色彩 | 纸白为主，黑色少量定位，金色只用于信号 |

这一块的目标是“长期使用不累，并且一眼知道哪里重要”。

它必须更克制、更工业、更有秩序。不能把入口站的黑色氛围直接搬进工作区，否则注意力会散。

## 2. 产品结构

疆域的结构不是“聊天框优先”，而是：

```text
全球信息海洋
  -> 我的疆域
    -> 当前模块
      -> 当前任务 / 当前对话 / 当前信号
```

设计上对应三层：

| 层级 | UI 承载 |
|---|---|
| 全球 / 疆域 | 顶部导航、品牌铭牌 |
| 疆域正在发生什么 | GYEN PULSE 信号带 |
| 当前站点周边发生什么 | 模块主体、模型状态、上下文面板 |

`GYEN PULSE` 和底部/侧边的上下文状态不是两套系统，而是同一条信号总线的不同缩放层级。

- 顶部 Pulse：广角，我的疆域里发生的一切。
- 模块状态：中景，当前板块正在处理什么。
- 当前卡片/对话：近景，此刻我站在哪里。

## 3. 设计演化

### V0：初始工作台

最早的疆域更像一个轻量 Web App：可用，但薄，缺少重量。问题主要是：

- 线条太碎。
- 黑色没有纪律。
- 金色像装饰，不像信号。
- 模块之间像不同人做的。
- Logo 有概念，但不够生动。

### V1：从 sgsyen 的 Macro Pulse 得到启发

`sgsyen.com` 的走马灯给了一个关键启发：信息流不是装饰，而是世界观的仪表盘。

疆域不需要抓取全世界的亿级信息。疆域需要经营自己的小池塘：

- 我今天推进什么。
- 哪些项目卡住。
- 哪些文件未提交。
- 哪些设计系统应冻结。
- 哪些信号已经腐烂。

因此 `Macro Pulse` 被转译为 `GYEN PULSE`。

### V2：Pulse 从装饰变成信号总线

早期 Pulse 太像一个中规中矩按钮。后来确定：

- 它必须是横向读数舱。
- 默认折叠为单行走马灯。
- 展开后是处理面板，不是说明文字。
- 每条信号必须具体，不能只写 `6 signals` 这种空摘要。
- 数字、单位、项目名必须有节奏，不可互相脱节。

### V3：注意力色度表

设计开始从“凭感觉”进入“色度纪律”。

核心原则：

```text
黑色 = 当前定位
仪表黑 = 系统读数
金色 = 信号
纸白 = 工作区
灰阶 = 注意力坡道
```

黑色太多，用户会无法判断“我在哪里”。金色太多，信号会贬值。灰阶不连续，页面会碎。

### V4：工作台凝实

命令轨道、按钮高度、边线、模块间距开始统一。

重要变化：

- `+ NEW` 和 `创意灵感` 保持同一按钮语言。
- 顶部导航不混入 Pulse，Pulse 独立成为第二层轨道。
- 模型状态从松散按钮，演化为系统状态抽屉。
- 中央空状态从欢迎语变为“待命工作台”。

### V5：老爷车 Logo 成立

Logo 的意境被确定：

人类有了汽车，生活半径从十公里扩展到更远的疆域。AI 对思想的作用也是这样，它不是一个聊天机器人，而是把人的思维半径推远。

所以疆域主 Logo 不是抽象科技图标，而是一台向疆域驶去的早期汽车。

### V6：模块级 VI 对齐

工作邮件、项目看板、复试账簿、军工密钥、会员中心开始按同一套 VI 收敛：

- 纸白底。
- 黑色只表示当前项或核心仪表。
- 低饱和状态色。
- 卡片与表格靠细微纸面差异建立层次，不靠粗框。
- 操作按钮保持矩形、短促、稳定。

### V7：9.5 分方向

后续演化方向不是堆内容，而是抠细节：

- 线条减少，结构更整体。
- 灰阶更连续。
- 动效更轻、更丝滑。
- 数字和单位更像同一组信号。
- Logo 更干净，透明背景，不能有图块感。
- 模块之间有个性，但不脱离疆域 VI。

### 4.1 基础 Token

| Token | 色值 | 用途 |
|---|---|---|
| `--gyen-n0-selected` | `#141414` | 当前选中、当前位置、当前模型 |
| `--gyen-n1-readout` | `#262420` | 仪表读数舱、Pulse 黑底 |
| `--gyen-n2-title` | `#3D3A34` | 品牌标题、主标题 |
| `--gyen-n3-content` | `#55514A` | 主要内容文字 |
| `--gyen-n4-label` | `#6B675E` | 标签、说明 |
| `--gyen-n5-control` | `#837E70` | 普通按钮文字、未选中控件 |
| `--gyen-n6-weak` | `#A6A196` | 弱信息、时间、占位 |
| `--gyen-n7-retreat` | `#C4BFB2` | 退场状态 |
| `--gyen-n8-hairline` | `#DEDAD0` | 结构线、分隔线 |
| `--gyen-n9-card` | `#ECE8DE` | 卡片基底、轨道 |
| `--gyen-n10-page` | `#F2EEE6` | 页面底色 |
| `--gyen-card-white` | `#FBFAF7` | 卡片白、输入框、普通面板 |
| `--gyen-gold-dark` | `#C9A227` | 主信号金 |
| `--gyen-gold-light` | `#A7853F` | 暗一档金，适合单位和收束 |
| `--gyen-gold-dim` | `#B8935E` | 次级金 |
| `--gyen-status-online` | `#5C7A5A` | 在线状态，低饱和绿 |
| `--gyen-status-unavailable` | `#B5AFA2` | 不可用状态，退场灰 |

### 4.2 注意力配额

| 颜色层 | 页面占比建议 | 规则 |
|---|---:|---|
| 选中黑 | 小于 5% | 只表示“你在这里” |
| 仪表黑 | 小于 8% | 只用于系统读数，不表示选中 |
| 信号金 | 小于 3% | 只用于数字、异常、待判断 |
| 纸白 / 卡白 | 大于 70% | 主体呼吸空间 |
| 灰阶 | 约 15% | 结构、标签、弱信息 |

### 4.3 黑色规则

黑色不是“重要”的泛用颜色。黑色只有两个语义：

1. 当前选中。
2. 仪表正在读数。

不能因为某个按钮想显得高级，就给黑底。否则页面会到处都是“当前项”，专注度会崩。

### 4.4 金色规则

金色不做装饰。金色只服务信号。

允许使用：

- Pulse 数字：`94`、`V2`、`7D`。
- 未处理数量：`03`。
- 待裁决竖线。
- 当前需要关注的状态点。

不允许使用：

- 普通标题。
- 普通按钮。
- 纯装饰边框。
- 大面积背景。

### 4.5 线条规则

疆域不靠粗线建立工业感。线条只负责结构。

- 细线必须低对比。
- 不要把区域切成一块一块。
- 大面板可以靠纸面色差、内阴影、间距建立层次。
- Logo 周围不能有框。

## 5. 字体系统

| 字体 | 用途 | 气质 |
|---|---|---|
| `Noto Serif SC` | `疆域` 中文品牌名 | 权威、碑铭、厚重 |
| `Cinzel` | `GSYEN` 英文品牌名 | 铭牌、古典、制度感 |
| `HarmonyOS Sans SC / Inter` | UI 正文 | 清晰、工作台 |
| `JetBrains Mono` | 数字、标签、状态、读数 | 仪表、精密、系统 |
| `iA Writer` 系列 | 写作与画布 | 安静、长期阅读 |

字体原则：

- 品牌可以有仪式感。
- 工作区必须清楚。
- 数字和状态必须有仪表感。
- 不用花哨字体制造高级感。

## 6. Logo 系统

### 6.1 主 Logo：疆域老爷车

主 Logo 是一台早期汽车。

它的语义：

- 汽车让人的行动半径扩大。
- AI 让人的思想半径扩大。
- 疆域就是这个扩展后的工作坊和领地。

设计规则：

- 车头朝左，用于品牌回看与出发感。
- 透明背景。
- 不放在方框里。
- 不做卡片底。
- 黑底页面使用白色版本。
- 纸白页面使用深色版本。
- 点击时可以有车头轻轻翘起的“启动感”。
- 不能重新画成玩具车、太空舱、卡通车。

当前资源：

| 文件 | 用途 |
|---|---|
| `public/brand/gsyen-frontier-car.png` | 原始深色版本 |
| `public/brand/gsyen-frontier-car-left-transparent.png` | 纸白工作区透明版本 |
| `public/brand/gsyen-frontier-car-left-white.png` | 黑底入口站白色版本 |

### 6.2 Logo 表面规则

Logo 周围必须透明。

不允许：

- 方形底板。
- 图片白边。
- 卡片框。
- 格子框。
- 被背景裁切。

允许：

- 直接落在纸白上。
- 直接落在黑底上。
- 用尺寸、位置和透明度融入环境。

### 6.3 模块 Logo

模块 Logo 用于告诉用户“当前在哪个疆域板块”。

| 模块 | 中文 | 英文/副标题 | 图形语义 |
|---|---|---|---|
| `chat` | 疆域灵阁 | GSYEN Muse | 主疆域，老爷车 |
| `mail` | 工作邮件 | Hermes | 私密邮件信道，信封 |
| `schedule` | 项目看板 | Flow | 信息流转，看板双栏 |
| `calendar` | 日程日历 | Chronos | 时间格栅，动态日期 |
| `finance` | 复试账簿 | Atelier Ledger | 资产复式记账，账簿 + 钱符 |
| `password` | 军工密钥 | Citadel | 密钥生成与保管，盾牌 |
| `brand` | PRISM实验 | Prism | 品牌基因折射，三角棱镜 |

规则：

- 顶部大 Logo 使用当前模块图形。
- 灵阁首页使用老爷车。
- 小导航图标可以更简化。
- 当前模块黑底选中。
- 非当前模块灰色退后。

## 7. 板块设计

### 7.1 疆域灵阁 / GSYEN Muse

核心定位：智能美学设计与品牌策略助手。

设计关键词：

- 中央待命。
- 信息流总线。
- 模型状态。
- 个人疆域入口。

关键组件：

- `GYEN PULSE`：疆域信号带。
- `Model Strip`：模型选择与状态。
- `ModelStatusPanel`：系统状态抽屉。
- `ChatEmptyState`：灵阁待命，而不是欢迎页。

### 7.2 工作邮件 / Hermes

核心定位：极雅私密邮件信道。

设计关键词：

- 封档。
- 收件阁。
- 星标函。
- 加密目录。

规则：

- 左侧目录当前项黑底。
- 邮件列表干净、横向分层。
- 安全状态使用弱灰和低饱和色，不用大红大绿。

### 7.3 项目看板 / Flow

核心定位：信息流转看板工作系统。

设计关键词：

- 流程。
- 卡片。
- 阶段。
- 低噪工作板。

规则：

- 看板列保持纸面感。
- 状态点降饱和。
- 卡片不做厚阴影。
- 当前拖动或选中才加深。

### 7.4 日程日历 / Chronos

核心定位：极速格栅日程空间。

设计关键词：

- 日期。
- 格栅。
- 执行周历。
- 单日重点。

规则：

- 时间信息要清楚，不要装饰化。
- 当前视图用黑色。
- 数量统计用小范围数字强调。

### 7.5 复试账簿 / Atelier Ledger

核心定位：奢雅资产复式记账账簿。

设计关键词：

- 仪表。
- 金额。
- 收支。
- 账目明细。

规则：

- 核心金额可进入黑色仪表卡。
- 盈利/支出用低饱和状态色。
- 表格以纸白为主，不做金融 App 式彩色面板。

### 7.6 军工密钥 / Citadel

核心定位：军事级密匙生成与保管箱。

设计关键词：

- 黑色仪表。
- 密钥强度。
- 凭证库。
- 本地安全。

规则：

- 生成器可以使用大面积读数黑。
- 右侧凭证列表保持纸白。
- 危险操作可以红，但红要小面积、低扩散。

### 7.7 PRISM实验 / Brand Lab

核心定位：品牌基因折射实验室。

子板块：

- 订单。
- 往来。
- 仓库。
- PRISM。
- 设计主画布。
- 周边物料效果。
- 穹弯算筹。
- 会员中心。

规则：

- PRISM 是品牌系统，不是普通后台。
- 子导航当前项用黑底。
- 表单和会员中心以纸白、细线、留白建立高级感。
- 状态徽章降饱和，不做明亮 SaaS 彩色标签。

## 8. 核心组件语法

### 8.1 顶部 Header

Header 承担两个任务：

1. 品牌铭牌。
2. 当前板块导航。

规则：

- Header 本身浅色，不与 Pulse 混在一起。
- 当前模块黑底。
- 右侧登录/注册是动作，不是当前位置，不能滥用黑色。
- 模块副标题在 Logo 区显示，强化当前板块身份。

### 8.2 Command Deck

命令轨道是工作台的第一层操作区。

包含：

- 折叠侧栏。
- 新建。
- 创意灵感。
- GYEN PULSE。
- Model Strip。

规则：

- 高度统一。
- 左侧按钮和 Pulse 不抢层级。
- Pulse 是横条，不是普通按钮。
- Model 区不能被 Pulse 挤压或遮挡。

### 8.3 GYEN PULSE

Pulse 是“疆域信号总线”。

折叠态：

- 单行横条。
- 内容滚动。
- 数字明确。
- 单位和数字同一家族色度。

展开态：

- 工作台，不是说明面板。
- 每条信号有具体事项。
- 右侧有动作。
- 不允许只显示摘要数字。

### 8.4 Model Status

模型状态不是装饰灯，而是系统状态抽屉。

状态包括：

- 当前模型。
- 模型 ID。
- 在线/离线。
- 上下文。
- 记忆。
- 工具就绪度。

规则：

- 不可用是退场灰，不是黑底警告。
- 在线绿必须低饱和。
- 检测中可使用信号金。

### 8.5 中央待命态

空状态不是欢迎语，而是“工作台待命”。

应该展示：

- 当前上下文。
- 今日焦点。
- 待裁决。
- 本地/同步状态。
- 输入意图。

不应该展示：

- 营销文案。
- 大幅插画。
- 教程式说明。

### 8.6 侧栏

侧栏是档案库，不是普通聊天列表。

规则：

- 当前项黑底或明确左标。
- 空状态要安静。
- 本地存档、团队、历史记录应该有仓库感。

### 8.7 卡片与面板

卡片用于重复内容、弹层、真实工具，不用于把每个页面段落都包起来。

原则：

- 卡片半径小。
- 阴影极轻。
- 结构靠纸面差异和间距。
- 不做玻璃拟态。

详细卡片锁见 `docs/GSYEN_90_DESIGN_LOCK.md`。

## 9. 动效规则

疆域的动效应该像机械阻尼，不像网页炫技。

允许：

- 150ms 到 240ms 的颜色和边框过渡。
- 抽屉展开使用短促 ease-out。
- 老爷车点击轻微抬头。
- Pulse 缓慢横向滚动。

不允许：

- 大幅弹跳。
- 彩色发光。
- 过度模糊。
- 频繁闪烁。

## 10. 禁止项

这些会破坏疆域设计：

- 大量黑底按钮。
- 大面积金色装饰。
- 蓝紫渐变。
- 圆角 SaaS 卡片堆叠。
- 玻璃拟态。
- 高饱和红绿蓝状态色。
- 把 Logo 放进方框。
- 细线过密，把页面切碎。
- 把信号写成空摘要。
- 给非当前项使用选中黑。
- 用“高级感”替代语义。

## 12. 后续演化路线

下一阶段不是继续加功能，而是继续提高一致性。

优先级：

1. **统一模块 Logo 品质**：每个模块图标都要像同一套仪器符号。
2. **减少无意义线条**：线条只服务结构，不制造碎片感。
3. **深化 Pulse 信息流**：接入真实项目、文件、任务、日程信号。
4. **建立上下文脉搏**：当前模块底部/侧边显示窄角信号。
5. **完善设计 Token 使用**：新组件必须先选 token，再写样式。
6. **形成截图评审标准**：每次 UI 改动用黑/金/纸白占比检查。
7. **桌面和 Web 一致**：Windows desktop 与 Web 使用同一套疆域语言，但保留桌面应用的窗口控制习惯。

## 13. 判断一句话

当一个设计不确定时，问四个问题：

1. 这个黑色是不是表示“我在这里”？
2. 这个金色是不是一个必须被看见的信号？
3. 这个线条是不是帮我理解结构，而不是切碎页面？
4. 这个模块是不是仍然属于“疆域”，而不是另一个产品？

如果四个问题都答得清楚，才继续做。
